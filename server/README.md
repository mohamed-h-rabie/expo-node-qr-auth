# 🛒 ecom-api

A high-performance, secure, and scalable E-commerce backend built with Node.js, Express, and MongoDB. This project serves as a comprehensive training ground for mastering the **MEN Stack**, covering everything from advanced authentication to complex database relationships and third-party payment integrations.

---

## 🚀 Key Modules & Features

### 🔐 Advanced Authentication System [Priority: Critical]
- **Secure Registration:** Hashed passwords using `bcrypt`.
- **OTP Verification:** 6-digit email verification via Gmail/Nodemailer.
- **Brute-Force Protection:** Intelligent limiting of verification attempts (Max 5).
- **JWT Authorization:** Role-based access control and session integrity.
- **Account Management:** Soft delete (deactivation), profile updates, and secure password resets.

### 📦 Product & Catalog Management [Priority: High]
- **Dynamic Catalog:** Full CRUD for Products and Categories.
- **Advanced Filtering:** Query-string based filtering (gte, gt, lte, lt), sorting, and pagination.
- **Search & Fields:** Select specific fields to optimize bandwidth.
- **Relational Data:** Products linked to Categories with Mongoose population.

### 🛍️ Shopping Experience
- **Payment Integration [Priority: High]:** Seamless checkout using **Paymob** (Auth, Order Creation, and Iframe Payment).
- **Cart System [Priority: Medium]:** Persistent cart management for authenticated users.
- **Wishlist [Priority: Low]:** Personal "Saved for Later" items.
- **Checkout & Bookings [Priority: High]:** Structured checkout flow and order management integrated with payment gateways.

### 🛡️ Security & Reliability
- **Password Hashing:** SHA-256 for OTPs and Bcrypt for passwords.
- **Environment Safety:** Centralized configuration via `env` variables.
- **Query Parsing:** Enhanced query string parsing via `qs`.

## ⚙️ Setup & Installation

### 1. Prerequisites
- Node.js installed
- MongoDB URI (Atlas or Local)
- Gmail account (for OTP service)
- **Paymob Account** (for payment integration)

### 2. Environment Variables
Create a `config.env` file in the root directory and add:
```env
PORT=3000
MONGO_URI=your_mongodb_connection_string
JWT_KEY=your_secret_key
JWT_EXPIRES=90d

# Email Service
GMAIL_USER=your_email@gmail.com
GMAIL_PASS=your_app_password

# Paymob Integration
PAYMOB_API_KEY=your_paymob_api_key
PAYMOB_INTEGRATION_ID=your_integration_id
PAYMOB_API_URL=https://accept.paymob.com/api
PAYMOB_FRAME_ID=your_iframe_id
```

### 3. Installation
```bash
# Install dependencies
npm install

# Run in development mode
npm run dev

# Run in production
npm start
```

## 🧪 Technical Deep Dive

### **Secure OTP Flow**
1. User signs up -> Random 6-digit OTP generated.
2. OTP is **hashed (SHA-256)** before storage.
3. Plain OTP sent via Gmail.
4. User submits OTP -> Stored hash compared using `crypto.timingSafeEqual` (prevents timing attacks).
5. Attempt counter limits user to 5 tries.

### **Advanced Query Engine**
The API supports professional-grade querying directly from the URL:
- `?price[gte]=100&sort=-price` (Filter products >= 100, sort descending).
- `?fields=name,price` (Return only name and price).
- `?page=2&limit=10` (Pagination).

### **Payment Integration (Paymob)**
The checkout process is a core feature that follows a secure 3-step handshake with the Paymob ecosystem:
1.  **Authentication:** The backend requests a short-lived `Auth Token` using the secret `PAYMOB_API_KEY`.
2.  **Order Registration:** A new order is created in Paymob's database with the specific `amount_cents` and currency, returning a unique `order_id`.
3.  **Payment Key Generation:** A payment token is generated by linking the `order_id` with user billing data (name, email, phone) and the `integration_id`.
4.  **Secure Handover:** The server returns a dynamic link to the **Paymob Iframe** (`iframe_id`), allowing the user to complete the transaction in a PCI-compliant environment.
5.  **Integration Point:** The logic is encapsulated in `bookingController.js` and exposed via the `/api/checkout/create-payment` endpoint.

## 📍 API Endpoints (Core)

### Users & Auth
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/api/users/signUp` | Register & send OTP |
| `POST` | `/api/users/signIn` | Login & get JWT |
| `POST` | `/api/users/verifyUser` | Verify OTP |

### Shop logic & Checkout
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/api/checkout` | Initialize Paymob payment flow |
| `GET` | `/api/cart` | Get user cart |
| `POST` | `/api/wishlist` | Add item to wishlist |

### Products & Categories
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/products` | Get all products (with filters/sort) |
| `POST` | `/api/products` | Create product (Admin) |
| `DELETE` | `/api/products/:id` | Delete product |
